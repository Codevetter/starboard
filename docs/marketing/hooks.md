# Starboard Landing Page Hook Variants

Five short hooks for GitHub starred repo intelligence.

## 1. Find The Good Stars

- Target audience: Developers with many GitHub stars.
- Pain: Your best saved repos are hard to find again.
- Promise: Starboard helps you search, sort, and group your stars.
- CTA: Connect GitHub and find one saved repo.

## 2. Your Stars Need A Map

- Target audience: Indie hackers and builders.
- Pain: You star useful tools, then lose them in a long list.
- Promise: Starboard turns that list into a simple map you can use.
- CTA: Try Starboard with your GitHub stars.

## 3. Stop Losing Useful Repos

- Target audience: Engineers who research tools often.
- Pain: A repo looked useful last month, but now you cannot find it.
- Promise: Starboard gives your starred repos search and clear filters.
- CTA: Search your stars in Starboard.

## 4. Know What You Saved

- Target audience: Founders and technical teams.
- Pain: GitHub stars pile up, but the why gets lost.
- Promise: Starboard helps show what each saved repo is for.
- CTA: Bring in your stars and review them.

## 5. A Smarter Star List

- Target audience: Developers picking libraries and tools.
- Pain: The GitHub star page is too flat for real decisions.
- Promise: Starboard helps you compare saved repos by topic and use.
- CTA: Start with your current stars.
